MASt3R-SLAM Loop Closure — L1 vs L2 Similarity

TUM RGB-D · ASMK Kernel · Trajectory Error Analysis

A targeted enhancement to MASt3R-SLAM's loop-closure module replaces the original Euclidean (L2) retrieval metric with a Manhattan (L1) distance, seamlessly integrating sum-of-absolute-differences into the ASMK kernel.

What Was Done

  • Kernel Modification: Swapped the dot-product calculation for a sum-of-absolute-differences operation, while preserving descriptor normalization and aggregation logic.
  • Automation Pipeline: Extended benchmarking scripts to run both L1 and L2 variants across nine TUM RGB-D sequences, capturing Absolute Trajectory Error (ATE) and frame-rate metrics.
  • Quantitative Analysis: Developed post-processing scripts to align estimated trajectories (via Horn's method) and compute per-sequence ATE, then plotted paired accuracy vs. throughput comparisons.

Key Findings

  • Accuracy Parity: Across all 9 sequences, the L1-based loop-closure matched L2's ATE within millimeter-scale differences — confirming negligible impact on localization quality.
  • Performance Trade-off: The L1 variant ran 25–30% slower than the hardware-optimized L2 implementation, highlighting the benefits of fused-multiply-add acceleration for Euclidean computations on modern CPUs.
  • Practical Takeaway: While Manhattan distance can serve as a drop-in replacement when hardware constraints limit vector-dot products, Euclidean metrics remain preferable for real-time loop-closure in high-throughput robotic applications.
