Scenario-Based AV Simulation Testing & Safety Metrics

Advised by Dr. Jeffery Wishart · CARLA Simulator · NHTSA Pre-Crash Scenarios

This framework implements all 37 NHTSA pre-crash test cases in the CARLA simulator — accounting for approximately 97% of light-duty vehicle crashes. Scenarios are defined via CARLA's Python API, instantiating traffic actors, environmental conditions, and sensor models to emulate real-world situations with high fidelity.

My Contributions

  • Scenario Database Development: Created a repository of NHTSA pre-crash scenarios, annotated with fidelity, relevance, and complexity scores aligned to regulatory guidelines.
  • Metric Standardization: Defined and computed Driving Assessment (DA) metrics — Complexity, Relevance, Fidelity, Metric, and Severity — using the OSA methodology to provide consistent safety profiles across tests.
  • Scenario Definition: Scripted each pre-crash case (rear-end, lane change, etc.) in CARLA to position vehicles, pedestrians, and environmental elements per NHTSA specifications.
  • Data Collection: Ran multiple simulation iterations per scenario while logging vehicle states, sensor outputs, and event triggers via CARLA's logging interface.
  • Metric Computation: Calculated safety metrics — minimum clearance distance, collision incident counts, response latency — using standardized formulas for safety assessment.

Key Safety Metrics

  • MDSE (Minimum Distance Safety Envelope): Measures closest distance maintained between the AV and other traffic participants.
  • OEDR Time (Operational Electronic Device Response): Captures latency from hazard detection to initiation of AV corrective action.
  • CI Count (Collision Incident): Records the number of collisions or near-miss events per scenario iteration.

Results

Preliminary validation shows the framework can identify safety-critical behaviors — such as insufficient clearance or excessive response latency — and quantify performance degradations as scenario complexity increases. By reducing reliance on real-world testing by up to 97%, this methodology accelerates development cycles and enhances reproducibility in CAV safety validation.
